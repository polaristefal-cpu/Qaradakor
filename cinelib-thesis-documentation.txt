1.1 НАЗВАНИЕ И ЦЕЛЬ ПРОЕКТА
---------------------------
Название: CineLib - Интеллектуальная библиотека фильмов

Цель: Разработать и реализовать полнофункциональное веб-приложение для 
управления персональной библиотекой просмотренных фильмов с интеграцией 
искусственного интеллекта для персонализированных рекомендаций, поиска и 
анализа.

1.2 АКТУАЛЬНОСТЬ ПРОЕКТА
------------------------
В современном мире объём доступного видеоконтента растёт экспоненциально, 
что создаёт проблему выбора для зрителей. Традиционные системы рекомендаций 
основаны на collaborative filtering, требующем большого количества пользователей. 
CineLib решает эту проблему, используя:

• Content-Based Filtering для персональных рекомендаций
• Искусственный интеллект (LLM) для естественного диалога о кино
• Sentiment Analysis для автоматического анализа эмоционального окраса отзывов
• AI-объяснения рекомендаций для прозрачности алгоритма

1.3 ОСНОВНЫЕ ВОЗМОЖНОСТИ
------------------------
✓ Система авторизации пользователей (Email/Password)
✓ Поиск фильмов через интеграцию с TMDB API (база данных более 500,000 фильмов)
✓ Управление персональной библиотекой просмотренных фильмов
✓ Система оценок (1-10 звёзд) и текстовых отзывов
✓ Персональные рекомендации на основе Content-Based Filtering
✓ AI-чатбот для поиска фильмов на естественном языке (GPT-4.1-mini)
✓ AI-анализ тональности отзывов (Sentiment Analysis)
✓ AI-объяснения "Почему вам понравится этот фильм"
✓ Система друзей с возможностью просмотра библиотек других пользователей
✓ Адаптивный современный UI с тёмной темой

1.4 ЦЕЛЕВАЯ АУДИТОРИЯ
---------------------
• Киноманы, ведущие учёт просмотренных фильмов
• Пользователи, ищущие качественные персональные рекомендации
• Люди, желающие делиться своими кинопредпочтениями с друзьями
• Исследователи, изучающие применение AI в рекомендательных системах

2.1 ТЕОРИЯ РЕКОМЕНДАТЕЛЬНЫХ СИСТЕМ
----------------------------------

2.1.1 Типы рекомендательных систем

A) COLLABORATIVE FILTERING (Коллаборативная фильтрация)
   Принцип: "Пользователям, похожим на вас, понравилось..."
   
   Преимущества:
   - Находит неожиданные связи
   - Не требует анализа контента
   
   Недостатки:
   - Проблема "холодного старта" (нужны данные от многих пользователей)
   - Невозможна для нового пользователя
   - Требует большую базу пользователей
   
   Примеры: Netflix (частично), Amazon

B) CONTENT-BASED FILTERING (Контентная фильтрация) ⭐ ИСПОЛЬЗУЕТСЯ В CINELIB
   Принцип: "Рекомендуем похожее на то, что вам нравилось"
   
   Преимущества:
   + Работает с единственным пользователем
   + Нет проблемы "холодного старта"
   + Прозрачность рекомендаций (можно объяснить, почему)
   + Немедленная персонализация
   
   Недостатки:
   - Риск "эхо-камеры" (рекомендации только похожего контента)
   - Требует качественные метаданные о контенте
   
   Примеры: Pandora Music Genome, YouTube (частично), Spotify

C) HYBRID SYSTEMS (Гибридные системы)
   Комбинируют оба подхода для устранения их недостатков.
   
   Примеры: Netflix (современная система), Spotify, Amazon


2.1.2 Математическая модель Content-Based Filtering в CineLib

Профиль пользователя U строится как вектор предпочтений:

U = {
  genreWeights: { [genreId]: normalizedScore },
  favoriteDirectors: Set<string>,
  favoriteActors: Set<string>
}

ГДЕ:
- genreWeights[g] = sum(rating/10 for each movie with genre g) / maxScore
- favoriteDirectors = directors appearing in ≥2 movies rated ≥7
- favoriteActors = top-5 actors from movies rated ≥7, appearing ≥2 times

Скоринг кандидата (candidate movie C):

score(C) = α₁·genreMatch + α₂·sourceWeight + α₃·popularity + 
           α₄·voteConfidence + α₅·crewBonus + α₆·trendBonus

ГДЕ:
• genreMatch = avg(genreWeights[g] for g in C.genres) ∈ [0,1]
• sourceWeight = rating(source_movie) / 10 ∈ [0,1]
• popularity = min(C.vote_average / 10, 1) ∈ [0,1]
• voteConfidence = min(C.vote_count / 100, 1) ∈ [0,1]
• crewBonus = 0.15 if director ∈ favoriteDirectors + 
              0.10 if actor ∈ favoriteActors ∈ [0, 0.25]
• trendBonus = min(log₁₀(max(C.popularity, 1)) / 3, 1) · 0.05 ∈ [0, 0.05]

ВЕСОВЫЕ КОЭФФИЦИЕНТЫ:
α₁ = 0.35  (жанровое соответствие — наиболее важно)
α₂ = 0.20  (оценка фильма-источника)
α₃ = 0.20  (качество/популярность)
α₄ = 0.10  (надёжность оценок)
α₅ = 1.00  (бонус за режиссёра/актёров)
α₆ = 1.00  (бонус за трендовость)

ПОРОГ ОТСЕЧЕНИЯ:
Рекомендуются только фильмы где score ≥ 0.45 И vote_average ≥ 5.5


2.2 ТЕОРИЯ ОБРАБОТКИ ЕСТЕСТВЕННОГО ЯЗЫКА (NLP)
----------------------------------------------

2.2.1 Large Language Models (LLM)

Определение: LLM — это нейронные сети, обученные на больших массивах текста
для понимания и генерации естественного языка.

Архитектура: Transformer (Attention Mechanism)
Ключевая формула Attention:

Attention(Q,K,V) = softmax(QK^T / √d_k) · V

ГДЕ:
Q - Query (запрос)
K - Keys (ключи)
V - Values (значения)
d_k - размерность ключей

Применение в CineLib:
• GPT-4.1-mini от OpenAI
• Контекстный диалог с учётом истории беседы
• Персонализация на основе библиотеки пользователя
• Структурированный вывод (JSON для рекомендаций)


2.2.2 Sentiment Analysis (Анализ тональности)

Определение: Автоматическое определение эмоциональной окраски текста.

Классы тональности:
• POSITIVE (позитивная) - восторг, удовлетворение
• NEGATIVE (негативная) - разочарование, критика
• MIXED (смешанная) - противоречивые эмоции
• NEUTRAL (нейтральная) - фактическое описание

Метрики:
• Sentiment Score ∈ [0.0, 1.0] где:
  0.0 = крайне негативно
  0.5 = нейтрально
  1.0 = крайне позитивно

Применение в CineLib:
• Автоматический анализ отзывов пользователей
• Извлечение ключевых эмоций
• Выявление ключевых слов, определяющих мнение


2.2.3 Prompt Engineering (Инженерия промптов)

Определение: Искусство формулирования запросов к LLM для получения 
желаемого результата.

Структура эффективного промпта:

SYSTEM PROMPT (системная роль):
1. Определение роли AI ("Ты — кино-эксперт")
2. Контекст (профиль пользователя)
3. Правила поведения
4. Формат вывода

USER PROMPT (запрос пользователя):
1. Конкретная задача
2. Релевантные данные
3. Примеры (few-shot learning)

Пример из CineLib (AI Explain):

```
SYSTEM: "Ты — кино-эксперт. Объясни коротко (2-3 предложения) 
почему этот фильм может понравиться пользователю, основываясь 
на его вкусах. Будь конкретным и убедительным. Отвечай на русском."

USER: "Фильм: 'Интерстеллар' (2014)
Жанры: Фантастика, Драма, Приключения
Режиссёр: Кристофер Нолан
Рейтинг TMDB: 8.4
Любимые фильмы пользователя: Начало (9/10); Марсианин (8/10); 
2001: Космическая Одиссея (10/10)"
```

Результат: Персонализированное объяснение с конкретной аргументацией.

2.3.2 RESTful API Design

Принципы REST:
1. Stateless (без сохранения состояния между запросами)
2. Единообразный интерфейс (стандартные HTTP методы)
3. Клиент-серверное разделение
4. Кэшируемость

HTTP методы в CineLib:

GET    /watched              - получить библиотеку
POST   /watched              - добавить фильм (с оценкой/отзывом)
DELETE /watched/:movieId     - удалить из библиотеки

GET    /friends              - список друзей
POST   /friends/request      - отправить запрос в друзья
POST   /friends/accept       - принять запрос
DELETE /friends/:friendId    - удалить из друзей

GET    /recommendations      - получить рекомендации

POST   /ai/chat              - AI-чат
POST   /ai/analyze-review    - анализ тональности отзыва
POST   /ai/explain           - AI-объяснение рекомендации

GET    /tmdb/*               - прокси к TMDB API


2.3.3 Authentication & Authorization

JWT (JSON Web Tokens):
Структура: header.payload.signature

Процесс авторизации в CineLib:

1. Регистрация: POST /signup
   → Supabase создаёт пользователя
   → Сохраняется profile в KV store

2. Вход: signInWithPassword()
   → Supabase возвращает JWT access_token
   → Токен сохраняется в клиенте

3. Защищённые запросы:
   Authorization: Bearer {SUPABASE_ANON_KEY}  ← для прохода через gateway
   x-user-token: {user_access_token}          ← для идентификации пользователя

4. Refresh: автоматическое обновление истекающих токенов
   Проверка: exp * 1000 < Date.now() + 60_000
   Действие: supabase.auth.refreshSession()

2.4.1 TMDB API (The Movie Database)

Описание: Открытая база данных фильмов, сериалов и актёров.

Ключевые эндпоинты:
• /search/movie         - поиск фильмов
• /movie/{id}           - детали фильма
• /movie/{id}/credits   - актёры и съёмочная группа
• /movie/{id}/recommendations - рекомендации TMDB
• /movie/{id}/similar   - похожие фильмы
• /trending/movie/week  - трендовые фильмы
• /movie/popular        - популярные фильмы

Формат данных (Movie object):
{
  id: number,
  title: string,
  original_title: string,
  overview: string,
  release_date: "YYYY-MM-DD",
  vote_average: number,    // 0-10
  vote_count: number,
  popularity: number,
  poster_path: string,     // /xyz.jpg
  backdrop_path: string,
  genre_ids: number[],
  genres: [{ id, name }],
  runtime: number,         // minutes
  credits: {
    cast: [{ name, character }],
    crew: [{ name, job }]
  }
}

Изображения:
Base URL: https://image.tmdb.org/t/p
Размеры: /w92, /w185, /w342, /w500, /w780, /original


2.4.2 OpenAI API (GPT-4.1-mini)

Эндпоинт: POST https://api.openai.com/v1/chat/completions

Запрос:
{
  model: "gpt-4.1-mini",
  messages: [
    { role: "system", content: "..." },
    { role: "user", content: "..." },
    { role: "assistant", content: "..." },
    ...
  ],
  temperature: 0.7,       // 0=детерминированность, 1=креативность
  max_tokens: 1024        // максимум токенов в ответе
}

Ответ:
{
  choices: [{
    message: {
      role: "assistant",
      content: "..."
    }
  }]
}

Стоимость (примерная):
- GPT-4.1-mini: ~$0.15 / 1M input tokens, ~$0.60 / 1M output tokens
  (экономичная модель для продакшена)

2.5.1 Principles of Modern UI/UX

1. CONSISTENCY (Согласованность)
   - Единые цвета, шрифты, компоненты

```typescript
// Автоматическая обработка JWT и refresh
async function getToken(): Promise<string> {
  const { data } = await supabase.auth.getSession();
  if (!data?.session) return publicAnonKey;
  
  const token = data.session.access_token;
  const payload = JSON.parse(atob(token.split('.')[1]));
  
  // Проверка истечения с буфером 60 секунд
  if (payload.exp * 1000 < Date.now() + 60_000) {
    const { data: refreshed } = await supabase.auth.refreshSession();
    return refreshed?.session?.access_token || publicAnonKey;
  }
  
  return token;
}

// Универсальный request wrapper
async function request(path: string, options: RequestInit = {}) {
  const isPublicRoute = path.startsWith("/tmdb/") || path === "/signup";
  const sessionToken = isPublicRoute ? null : await getToken();
  
  const headers = {
    "Content-Type": "application/json",
    "Authorization": `Bearer ${publicAnonKey}`,  // Для Supabase gateway
    ...(sessionToken && { "x-user-token": sessionToken })  // Для пользователя
  };
  
  const res = await fetch(`${BASE}${path}`, { ...options, headers });
  
  // Retry при 401 с refresh
  if (res.status === 401 && sessionToken) {
    const { data: refreshed } = await supabase.auth.refreshSession();
    if (refreshed?.session?.access_token) {
      // Повтор запроса с новым токеном
      return fetch(`${BASE}${path}`, {
        ...options,
        headers: { ...headers, "x-user-token": refreshed.session.access_token }
      }).then(r => r.json());
    }
  }
  
  return res.json();
}
```

```typescript
  // Токен пользователя передаётся через кастомный заголовок
```

```typescript
app.get("/make-server-59141208/tmdb/*", async (c) => {
  const apiKey = tmdbKey();
  
  // Извлечение пути после /tmdb/
  let path = c.req.path.replace("/make-server-59141208/tmdb", "");
  
  // Пробрасывание query параметров от клиента
  const reqUrl = new URL(c.req.url);
  const params = new URLSearchParams();
  reqUrl.searchParams.forEach((v, k) => params.set(k, v));
  const qs = params.toString();
  if (qs) path += (path.includes("?") ? "&" : "?") + qs;
  
  // Запрос к TMDB
  const url = `https://api.themoviedb.org/3${path}${
    path.includes("?") ? "&" : "?"
  }api_key=${apiKey}`;
  
  const resp = await fetch(url);
  const data = await resp.json();
  
  return c.json(data, resp.status as any);
});
```

3.3.6 Content-Based Recommendations (Алгоритм)

ПОЛНАЯ РЕАЛИЗАЦИЯ:

```typescript
app.get("/make-server-59141208/recommendations", async (c) => {
  const user = await getUser(c);
  if (!user) return c.json({ error: "Unauthorized" }, 401);
  
  try {
    const watched: any[] = await kv.get(`user:${user.id}:watched`) || [];
    
    // Если библиотека пуста, показываем популярные
    if (watched.length === 0) {
      const resp = await tmdbFetch("/movie/popular?language=ru-RU");
      const data = await resp.json();
      return c.json(data.results?.slice(0, 20) || []);
    }

    // --- ЭТАП 1: Построение профиля пользователя ---
    
    // Загрузка детальной информации о всех просмотренных фильмах
    const movieDetails = new Map<number, any>();
    await Promise.all(
      watched.map(async (w: any) => {
        try {
          const resp = await tmdbFetch(
            `/movie/${w.movieId}?language=ru-RU&append_to_response=credits`
          );
          const d = await resp.json();
          if (d.id) movieDetails.set(w.movieId, d);
        } catch {}
      })
    );

    // Подсчёт весов жанров
    const genreScores: Record<number, { 
      score: number; 
      name: string; 
      count: number 
    }> = {};
    
    // Подсчёт режиссёров и актёров из высоко оцененных фильмов
    const directorCounts: Record<string, number> = {};
    const actorCounts: Record<string, number> = {};

    for (const w of watched) {
      const detail = movieDetails.get(w.movieId);
      if (!detail) continue;
      
      const normalizedRating = (w.rating || 5) / 10;  // 0.0 - 1.0

      // Накопление баллов жанров
      for (const g of (detail.genres || [])) {
        if (!genreScores[g.id]) {
          genreScores[g.id] = { score: 0, name: g.name, count: 0 };
        }
        genreScores[g.id].score += normalizedRating;
        genreScores[g.id].count += 1;
      }

      // Подсчёт режиссёров и актёров из фильмов с оценкой ≥7
      if ((w.rating || 0) >= 7) {
        const directors = detail.credits?.crew?.filter(
          (cr: any) => cr.job === "Director"
        ) || [];
        for (const d of directors) {
          directorCounts[d.name] = (directorCounts[d.name] || 0) + 1;
        }
        
        const topActors = detail.credits?.cast?.slice(0, 5) || [];
        for (const a of topActors) {
          actorCounts[a.name] = (actorCounts[a.name] || 0) + 1;
        }
      }
    }

    // Нормализация весов жанров в диапазон [0, 1]
    const maxGenreScore = Math.max(
      ...Object.values(genreScores).map(g => g.score), 
      1
    );
    const genreWeights: Record<number, number> = {};
    for (const [gid, g] of Object.entries(genreScores)) {
      genreWeights[Number(gid)] = g.score / maxGenreScore;
    }

    // Любимые режиссёры и актёры (встречаются ≥2 раза)
    const favDirectors = new Set(
      Object.entries(directorCounts)
        .filter(([, c]) => c >= 2)
        .map(([n]) => n)
    );
    const favActors = new Set(
      Object.entries(actorCounts)
        .filter(([, c]) => c >= 2)
        .map(([n]) => n)
    );

    console.log("[Recs] Genre weights:", JSON.stringify(genreScores));
    console.log("[Recs] Fav directors:", [...favDirectors]);
    console.log("[Recs] Fav actors:", [...favActors].slice(0, 10));

    // --- ЭТАП 2: Сбор кандидатов ---
    
    const watchedIds = new Set(watched.map(w => w.movieId));
    
    // Берём топ-8 оценённых фильмов
    const topRated = [...watched]
      .sort((a, b) => (b.rating || 0) - (a.rating || 0))
      .slice(0, 8);
    
    const candidateMap = new Map<number, { 
      movie: any; 
      sourceRating: number 
    }>();

    // Для каждого топового фильма запрашиваем recommendations и similar
    await Promise.all(
      topRated.map(async (w: any) => {
        try {
          const [recResp, simResp] = await Promise.all([
            tmdbFetch(`/movie/${w.movieId}/recommendations?language=ru-RU`),
            tmdbFetch(`/movie/${w.movieId}/similar?language=ru-RU`),
          ]);
          
          const recData = await recResp.json();
          const simData = await simResp.json();
          const allResults = [
            ...(recData.results || []), 
            ...(simData.results || [])
          ];
          
          for (const r of allResults) {
            if (!watchedIds.has(r.id) && !candidateMap.has(r.id)) {
              candidateMap.set(r.id, { 
                movie: r, 
                sourceRating: w.rating || 5 
              });
            }
          }
        } catch {}
      })
    );

    // --- ЭТАП 3: Скоринг кандидатов ---
    
    // Загрузка credits для кандидатов (для проверки режиссёра/актёров)
    const candidateIds = [...candidateMap.keys()];
    const creditMap = new Map<number, any>();
    
    await Promise.all(
      candidateIds.slice(0, 60).map(async (id) => {
        try {
          const resp = await tmdbFetch(
            `/movie/${id}?language=ru-RU&append_to_response=credits`
          );
          const d = await resp.json();
          if (d.id) creditMap.set(id, d);
        } catch {}
      })
    );

    const scored: { 
      movie: any; 
      score: number; 
      breakdown: any 
    }[] = [];

    for (const [id, { movie, sourceRating }] of candidateMap) {
      const detail = creditMap.get(id);

      // 1. Genre Match (0..1)
      const movieGenreIds = 
        movie.genre_ids || detail?.genres?.map((g: any) => g.id) || [];
      let genreMatch = 0;
      if (movieGenreIds.length > 0) {
        const sum = movieGenreIds.reduce(
          (acc: number, gid: number) => acc + (genreWeights[gid] || 0), 
          0
        );
        genreMatch = sum / movieGenreIds.length;
      }

      // 2. Source Weight (0..1)
      const sourceWeight = sourceRating / 10;

      // 3. Popularity / Quality (0..1)
      const popularity = Math.min((movie.vote_average || 0) / 10, 1);

      // 4. Vote Confidence (0..1)
      const voteConfidence = Math.min((movie.vote_count || 0) / 100, 1);

      // 5. Crew Bonus (0..0.25)
      let crewBonus = 0;
      if (detail?.credits) {
        const dirs = detail.credits.crew?.filter(
          (cr: any) => cr.job === "Director"
        ) || [];
        if (dirs.some((d: any) => favDirectors.has(d.name))) {
          crewBonus += 0.15;
        }
        
        const actors = detail.credits.cast?.slice(0, 5) || [];
        if (actors.some((a: any) => favActors.has(a.name))) {
          crewBonus += 0.10;
        }
      }

      // ФИНАЛЬНАЯ ФОРМУЛА:
      const finalScore =
        (genreMatch * 0.35) +
        (sourceWeight * 0.20) +
        (popularity * 0.20) +
        (voteConfidence * 0.10) +
        crewBonus +
        (Math.min(
          Math.log10(Math.max(movie.popularity || 1, 1)) / 3, 
          1
        ) * 0.05);

      // Порог качества: score ≥ 0.45 И vote_average ≥ 5.5
      if (finalScore >= 0.45 && (movie.vote_average || 0) >= 5.5) {
        scored.push({
          movie: { ...movie, _score: Math.round(finalScore * 100) },
          score: finalScore,
          breakdown: { 
            genreMatch, 
            sourceWeight, 
            popularity, 
            voteConfidence, 
            crewBonus 
          },
        });
      }
    }

    // Сортировка по score (убывание)
    scored.sort((a, b) => b.score - a.score);

    console.log(
      `[Recs] ${candidateMap.size} candidates → ` +
      `${scored.length} passed threshold, returning top 20`
    );
    
    if (scored.length > 0) {
      console.log("[Recs] Top 3:", scored.slice(0, 3).map(s => ({
        title: s.movie.title,
        score: s.score.toFixed(3),
        ...s.breakdown,
      })));
    }

    return c.json(scored.slice(0, 20).map(s => s.movie));
    
  } catch (e: any) {
    console.log("[Recs] Error:", e.message);
    return c.json({ error: `Recommendations error: ${e.message}` }, 500);
  }
});
```

```typescript
async function callOpenAI(
  messages: any[], 
  temperature = 0.7, 
  max_tokens = 1024
) {
  const resp = await fetch("https://api.openai.com/v1/chat/completions", {
    method: "POST",

      Authorization: `Bearer ${openaiKey()}`,

      messages,
      temperature,
      max_tokens,
    }),
  });
  
  const data = await resp.json();
  if (data.error) throw new Error(`OpenAI error: ${data.error.message}`);
  
  return data.choices[0].message.content;
}
```

```
KEY: user:{userId}:profile
VALUE: {

KEY: user:{userId}:friends
VALUE: [userId1, userId2, ...]

KEY: user:{userId}:friend_requests
VALUE: [
  {
    fromId: string,
    fromName: string,
    fromEmail: string
  },
  ...
]
```

```typescript
import * as kv from "./kv_store.tsx";
```

5.3 SECURITY CONSIDERATIONS (Безопасность)
-------------------------------------------

A) AUTHENTICATION FLOW:

   Registration:
   1. User → Frontend: email, password, name
   2. Frontend → Backend: POST /signup
   3. Backend → Supabase Admin: createUser()
   4. Supabase создаёт пользователя с email_confirm: true
   5. Backend сохраняет profile в KV
   6. Response: user object (без пароля)

   Login:
   1. User → Frontend: email, password
   2. Frontend → Supabase Client: signInWithPassword()
   3. Supabase возвращает: { session: { access_token, refresh_token } }
   4. Frontend сохраняет session в Context
   5. Все последующие запросы включают access_token в x-user-token

   Token Refresh:
   - Автоматическая проверка expiry перед каждым запросом
   - Если exp < now + 60s → автоматический refresh
   - При 401 → повторный refresh и retry запроса

B) API KEY PROTECTION:

   TMDB API Key:
   ❌ НЕ хранится в frontend коде
   ✓ Хранится в Supabase Environment Variables
   ✓ Используется только на сервере
   ✓ Проксируется через /tmdb/* endpoints

   OpenAI API Key:
   ❌ НЕ доступен клиенту
   ✓ Хранится в Supabase Environment Variables
   ✓ Все AI запросы через backend endpoints

C) AUTHORIZATION:

   Public Endpoints (без авторизации):
   - POST /signup
   - GET /tmdb/*  (TMDB proxy)

   Protected Endpoints (требуют JWT):
   - GET /profile
   - GET/POST/DELETE /watched
   - GET/POST/DELETE /friends/*
   - GET /recommendations
   - POST /ai/*

   Проверка авторизации:
   ```typescript
   const user = await getUser(c);
   if (!user) return c.json({ error: "Unauthorized" }, 401);
   ```

D) DATA ISOLATION:

   Каждый пользователь имеет доступ только к своим данным:
   - user:{userId}:watched
   - user:{userId}:friends
   - user:{userId}:friend_requests

   Доступ к данным друга только если:
   - friendId in user.friends list
   - Эндпоинт: GET /friends/:friendId/watched

E) CORS POLICY:

   ```typescript
   cors({
     origin: "*",  // В продакшене: указать конкретный домен
     allowHeaders: ["Content-Type", "Authorization", "x-user-token"],
     allowMethods: ["GET", "POST", "PUT", "DELETE", "OPTIONS"],
   })
   ```
